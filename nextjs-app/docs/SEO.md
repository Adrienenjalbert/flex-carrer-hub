# SEO Guide

## Overview

The Indeed Flex Career Hub is designed with SEO as a core feature. This document covers all SEO implementations and best practices.

---

## Architecture

### Static Generation (SSG)

All pages are pre-rendered at build time via `generateStaticParams`:

```typescript
// Generates static HTML for each role
export function generateStaticParams() {
  return roles.map((role) => ({
    roleSlug: role.slug,
  }));
}
```

**Benefits:**
- Instant page loads (no server rendering delay)
- Full HTML for search engine crawlers
- Better Core Web Vitals scores
- Lower hosting costs

## Structured Data (JSON-LD)

### Available Schemas

| Schema | Use Case |
|--------|----------|
| `FAQSchema` | FAQ sections (triggers rich snippets) |
| `ArticleSchema` | Blog/guide content |
| `WebPageSchema` | General pages with breadcrumbs |
| `BreadcrumbSchema` | Navigation breadcrumbs |
| `JobPostingSchema` | Job listings |
| `HowToSchema` | Step-by-step guides |
| `LocalBusinessSchema` | Location pages |

### Implementation

```tsx
import { FAQSchema, WebPageSchema, BreadcrumbSchema } from "@/components/career-hub/seo";

export default function Page() {
  const faqs = [
    { question: "How much does X cost?", answer: "X costs..." },
    { question: "How long does it take?", answer: "It takes..." },
  ];

  return (
    <>
      {/* Schema markup - renders in <head> as script tags */}
      <WebPageSchema
        name="Page Title"
        description="Page description"
        url="https://indeedflex.com/page"
        breadcrumb={[
          { name: "Home", url: "https://indeedflex.com" },
          { name: "Current Page" },
        ]}
      />
      <FAQSchema questions={faqs} />
      <BreadcrumbSchema
        items={[
          { name: "Home", url: "https://indeedflex.com" },
          { name: "Current Page" },
        ]}
      />
      
      {/* Page content */}
      <main>...</main>
    </>
  );
}
```

## Sitemap

### Split Sitemap Architecture

The sitemap uses Next.js `generateSitemaps()` to produce 14 split sitemaps for better crawlability:

```
/sitemap/core.xml           — hub pages, about, tools index, guides index
/sitemap/roles.xml           — all role pages
/sitemap/cities.xml          — city + location pages
/sitemap/city-roles.xml      — high-value city×role combinations
/sitemap/tools.xml           — all calculator/tool pages
/sitemap/guides.xml          — how-to-become, interview, certification guides
/sitemap/articles.xml        — guide articles, financial tips
/sitemap/states.xml          — paycheck + unemployment per state
/sitemap/job-application.xml — toolkit, templates, cover letters, resume examples
/sitemap/personas.xml        — persona hubs, pay ranges, schedules
/sitemap/seasonal.xml        — seasonal hiring, events
/sitemap/wage-report.xml     — annual wage report pages
/sitemap/career-evaluations.xml
/sitemap/salary-by-city.xml
```

Source: `src/app/sitemap.ts`

### Accessing Sitemaps

- Sitemap index: `/sitemap.xml` (auto-generated by Next.js)
- Format: XML with `<sitemapindex>` pointing to 14 child sitemaps
- Total URLs: 500+

## URL Structure

### Best Practices Applied

| Pattern | Example | Benefit |
|---------|---------|---------|
| Descriptive slugs | `/bartender` not `/role/123` | Keyword-rich URLs |
| Hierarchical paths | `/career-hub/roles/bartender` | Clear site structure |
| Lowercase | `/new-york` not `/New-York` | Consistency |
| Hyphens | `/new-york` not `/new_york` | SEO standard |

### Current URL Patterns

```
/                                    # Homepage
/career-hub                          # Hub landing
/career-hub/roles                    # All roles
/career-hub/roles/[roleSlug]         # Role detail
/career-hub/cities/[citySlug]        # City detail
/career-hub/locations/[locationSlug] # Location detail
/paycheck-calculator/[stateSlug]     # State calculator
/unemployment-benefits/[stateSlug]   # State UI guide
/certifications/[slug]               # Certification guide
/how-to-become/[roleSlug]            # Career path
/interview-questions/[roleSlug]      # Interview prep
```

## Content SEO

### Page Structure

Every page follows this SEO-optimized structure:

```tsx
<>
  {/* 1. Schema markup */}
  <FAQSchema questions={faqs} />
  
  {/* 2. Breadcrumb navigation */}
  <nav aria-label="Breadcrumb">
    <Link href="/career-hub">Career Hub</Link>
    <span>›</span>
    <span>Current Page</span>
  </nav>
  
  {/* 3. Main heading (H1 - only one per page) */}
  <h1>Primary Keyword in Title</h1>
  
  {/* 4. Introduction paragraph */}
  <p>Engaging intro with target keywords...</p>
  
  {/* 5. Subheadings (H2, H3) */}
  <h2>Related Topic</h2>
  <p>Content...</p>
  
  {/* 6. FAQ section (triggers rich snippets) */}
  <section>
    <h2>Frequently Asked Questions</h2>
    {faqs.map((faq) => (
      <div key={faq.question}>
        <h3>{faq.question}</h3>
        <p>{faq.answer}</p>
      </div>
    ))}
  </section>
  
  {/* 7. Internal links */}
  <section>
    <h2>Related Topics</h2>
    <Link href="/related-page">Related content</Link>
  </section>
</>
```

### Heading Hierarchy

```
H1: Page Title (one per page)
├── H2: Major Section
│   ├── H3: Subsection
│   └── H3: Subsection
├── H2: Another Section
│   ├── H3: Subsection
│   └── H3: Subsection
└── H2: FAQ Section
    ├── H3: Question 1
    └── H3: Question 2
```

## Internal Linking

### Strategy

1. **Contextual links**: Link relevant terms to related pages
2. **Related content sections**: "Other Roles", "Nearby Cities"
3. **Breadcrumbs**: Clear navigation path
4. **Footer links**: Key pages always accessible

### Implementation

```tsx
// In role pages
<section>
  <h2>Related Roles</h2>
  {relatedRoles.map((role) => (
    <Link href={`/career-hub/roles/${role.slug}`}>
      {role.title}
    </Link>
  ))}
</section>

// In city pages
<section>
  <h2>Popular Roles in {city.name}</h2>
  {roles.slice(0, 6).map((role) => (
    <Link href={`/career-hub/roles/${role.slug}`}>
      {role.title} Jobs
    </Link>
  ))}
</section>
```

## Performance (Core Web Vitals)

### Current Optimizations

| Metric | Target | Implementation |
|--------|--------|----------------|
| LCP (Largest Contentful Paint) | < 2.5s | Static generation, optimized images, `priority` on hero images |
| INP (Interaction to Next Paint) | < 200ms | Minimal client-side JS, deferred non-critical scripts |
| CLS (Cumulative Layout Shift) | < 0.1 | Reserved space for dynamic content, font `display: swap` |
| TTFB (Time to First Byte) | < 600ms | CDN delivery of static pages |

Note: FID was replaced by INP (Interaction to Next Paint) as a Core Web Vital in March 2024. INP measures responsiveness across all interactions, not just the first.

### Image Optimization

```tsx
import Image from "next/image";

<Image
  src="/images/hero.jpg"
  alt="Descriptive alt text with keywords"
  width={800}
  height={600}
  priority  // For above-the-fold images

```

## Answer Engine Optimization (AEO)

In addition to traditional SEO, the Career Hub optimizes for AI-powered answer engines (ChatGPT, Perplexity, Google AI Overviews, Gemini). See `BRAND.md` for full AEO content rules.

### Technical AEO Requirements

| Requirement | Implementation |
|-------------|---------------|
| **Answer-first content** | 40-60 word definitive answer in first paragraph of every article |
| **Question-format H2s** | H2s phrased as questions matching PAA queries |
| **FAQPage schema** | JSON-LD with `mainEntity` Q&A pairs |
| **Article schema** | Add `speakable` property marking citable sections |
| **Entity references** | Add `about` and `mentions` to schemas for semantic clarity |
| **Fresh timestamps** | `dateModified` always current and ISO 8601 format |
| **AI crawler access** | Allow `OAI-SearchBot`, `PerplexityBot`, `ClaudeBot`, `GoogleOther` in `robots.ts` |

### Search Console Integration

Use the `search-console-metrics` Supabase Edge Function to pull per-page performance data:

```typescript
const response = await fetch(
  `${SUPABASE_URL}/functions/v1/search-console-metrics`,
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${SUPABASE_ANON_KEY}`,
    },
    body: JSON.stringify({
      urlFilter: '/career-hub/',
      startDate: '2026-01-01',
      endDate: '2026-03-16',
    }),
  }

const { success, rows } = await response.json();
// rows: [{ keys: [url], clicks, impressions, ctr, position }]
```

### Content Engine Quality Scoring

The content engine (`src/lib/content-engine/`) provides automated scoring:

- `quality-scorer.ts` — scores articles across Brand (25%), CRAFT (25%), SEO (20%), AEO (15%), Depth (15%)
- `performance-tracker.ts` — tracks per-page health scores combining organic performance + quality + freshness
- `decay-detector.ts` — auto-detects content decay and queues refresh actions
- `strategy-learnings.ts` — extracts patterns from top-performing content

Run `npm run audit:quality` to score all articles.
